## Findings (Severity-Ranked)

1. **[BLOCKER] File deletion timeline is vague** (lines 514-515, 535) - The page states "client work does not sit on a server forever" and "the link stops working" but never explicitly says WHEN files are deleted from servers. Explicit language required: "Files are permanently deleted [X days] after upload or when the link expires, whichever comes first." This is load-bearing for privacy trust. Suggested fix: Add to hero section or "Why Ferry" card with exact timeline.

2. **[MAJOR] No privacy policy or data processing terms linked on landing page** (footer area, lines 615-632) - The footer shows a demo disclaimer but no links to privacy policy, terms, or GDPR/compliance information. Privacy-conscious users expect these signals. Suggested fix: Add "Privacy Policy" and "Terms" links to footer nav; consider a privacy badge or "GDPR compliant" statement near hero.

3. **[MAJOR] Server location and data residency not disclosed anywhere** (entire page) - A privacy-conscious user evaluates whether data stays in their jurisdiction (EU, US, etc.). The page never mentions where files are stored. Suggested fix: Add to "Why Ferry" section: "All files stored in [US/EU]-based data centers with [encryption type] encryption."

4. **[MEDIUM] "No account" claim is misleading** (line 539, value prop card) - Card states "No account to grab it" in context of recipients, but senders presumably need accounts for Pro tier. The narrowly-true claim risks being perceived as deceptive. Suggested fix: Clarify to "Recipients need no account—just a link." Separate sender account info to Pro pricing section.

5. **[MINOR] Reassurance copy undersells the privacy angle** (line 520) - "You set the limit. Need it longer or want to resend, you stay in control" is good but doesn't mention data deletion or retention periods. Suggested fix: Append "...and files are automatically deleted when they expire."

## Accessibility (WCAG 2.1 AA)

- **Contrast:** FAIL on `.expiry-tag` (lines 290-304). Text color `#7a4d05` on background `rgba(242, 165, 65, 0.16)` at 16% opacity yields insufficient contrast ratio against the light amber background. Fails WCAG AA 4.5:1 for normal text. Suggested fix: Increase opacity of background or darken text color to meet 4.5:1 minimum.

- **Touch targets:** PASS. Primary buttons (`.btn-primary`, `.btn-ghost`, lines 118-129) have 14px vertical padding + 26px horizontal, meeting 44x44px minimum. Modal close button (line 639) is 24px font-size line-height, sufficient.

- **Keyboard reachability of primary flow:** PARTIAL FAIL. The modal close button (`.modal-close`, line 360-369) has no visible focus styles defined. When Tab reaches it, keyboard users see no focus indicator. Suggested fix: Add `.modal-close:focus-visible { outline: 2px solid var(--teal); outline-offset: 2px; }` to CSS.

- **Alt text and focus visibility:** Modal uses `aria-modal="true"` and `aria-labelledby="sendTitle"` (line 635), good. But `.modal-close` button (line 639) has `aria-label="Close"`, which is correct, yet lacks visual focus feedback. Icons in value props (lines 533, 538, 543) use HTML entities (hourglass, lock, star) without alt text; OK as decorative, but confirm intent. Focus visibility is the blocker here.

## Persona Verdict

As a privacy-conscious user who has been burned before by services that retain data indefinitely: I would **not buy** and would **abandon at hero**. 

The expiring link framing actually works (changed me from nervous to cautiously interested), and the "no account for recipients" is attractive. But the page omits three critical trust signals a privacy-focused buyer needs:
1. Explicit file deletion timeline (not "forever")
2. Privacy policy and legal accountability
3. Data residency transparency

**One change that would move me from bounce to buy:** Add a single sentence to the hero reassurance line (line 520): "You set the limit. Files are automatically deleted when they expire. See our privacy policy to learn how we handle your data."
